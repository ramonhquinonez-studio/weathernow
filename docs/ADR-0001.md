# ADR-0001: MVVM + Clean + Inyección por inicializador

## Contexto
SwiftUI-first en iOS. Se busca testabilidad y bajo acoplamiento.

## Decisión
- MVVM para presentación.
- UseCases/Repositories para Domain/Data.
- Protocols + inyección por inicializador (sin singletons).
- Carpetas por módulos (Core/Networking/Persistence/Features).

## Consecuencias
- Testeo de ViewModels con stubs.
- Intercambio de repos (API vs. cache) sin tocar vistas.
